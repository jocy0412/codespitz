State Control Memory Address Pointer Variables Dispatch 참조의 참조를 왜 하는가? - Directive Reference 참조 참조와 변수안에 있는 주소값을 가져온다. 참조는 참조로 할당할 때 해당 참조용 변수를 가져오는 것이 아니라 참조와 변수 안에 들어있던 주소값을 가져온다. 1. 직접참조 a = "TEST" a주소 = 00, a크기 4 b = &a (b에는 값이 들어있는 것이 아니라 a의 주소를 참조하는 값이 있다. 이것을 참조라고 한다.) c = &b (b에 있는 참조값을 불러온다.) a = b & b = c (a = c) (현재는 a = b = c 가 맞다.) d = "ABC" b = "&d" (하지만 이렇게 되면 아래와 같은 문제가 생기게 된다.) b = d & b!=c (a = c) 이렇게 직접 참조하면 관리 하기가 어렵게 된다. 2. 간접참조 a = "TEST" a주소 = 00, a크기 4 b = {target:&a} c = &b d = "ABC" b.target = &d c.target == &d 자료구조를 크게 2개로 볼 수있다. 1. 메모리를 통으로 쓰는 배열 2. 링크드 리스트의 형식을 빌린 링크드 리스트 링크드 리스트의 원리는 우리가 현재의 상태를 관리하는 가장 큰 무기다. 객체지향은 .을 많이 사용한다.
